Integritetspolicy

Här förklaras hur personuppgifter behandlas via webbutvecklare.se. Poängen är enkel: jag samlar in det som behövs för att kunna svara, hjälpa dig och genomföra uppdrag inom hemsidor, uppdateringar och större webblösningar. Inte mer än nödvändigt.

Läs policyn Kontakta mig

Kort version

När du kontaktar mig via formulär eller mejl behandlas dina uppgifter för att jag ska kunna svara dig, hantera förfrågan och vid behov genomföra ett uppdrag. Uppgifter delas inte i onödan och sparas inte längre än det finns ett rimligt skäl till. Om du blir kund kan viss information även behöva behandlas för att bygga, uppdatera eller lämna över hemsidor och webblösningar på ett tydligt sätt.

1. Personuppgiftsansvarig

Personuppgiftsansvarig för behandling av personuppgifter via webbutvecklare.se är Nordlöf Nordic, med Erica Nordlöf som kontaktperson.

Har du frågor om hur uppgifter behandlas är du välkommen att kontakta mig direkt på erica@webutvecklare.se.

2. Vilka uppgifter som samlas in

När du använder formulär på sidan eller kontaktar mig via e-post kan följande uppgifter behandlas, beroende på vad du själv skickar in:

  • ✓namn
  • ✓e-postadress
  • ✓telefonnummer
  • ✓företagsnamn eller verksamhetsnamn
  • ✓ort eller område
  • ✓uppgifter om ditt uppdrag, din brief eller dina önskemål
  • ✓annan information som du själv väljer att skriva i formulär eller mejl
  • ✓i vissa uppdrag även material, inloggningsuppgifter eller annan teknisk information som behövs för att bygga, uppdatera eller lämna över en hemsida eller webblösning

Skicka helst inte känsliga personuppgifter i formulär eller vanlig e-post om det inte verkligen behövs. Det blir sällan bättre bara för att ett mejl innehåller mer dramatik.

3. Varför uppgifterna används

Personuppgifter används för att kunna:

  • ✓svara på frågor och förfrågningar
  • ✓ta fram demo, offert eller upplägg
  • ✓genomföra beställda uppdrag
  • ✓kommunicera om pågående arbete, ändringar, leverans eller betalning
  • ✓hantera administration, bokföring och affärsdokumentation när det behövs
  • ✓vid behov hantera överlämning av material, inloggningsuppgifter och koder efter köp om du senare vill fortsätta själv eller med någon annan

Uppgifterna används alltså för praktiskt kundarbete och normal företagsadministration. Inte för att bygga något avancerat personprofilscirkus.

4. Rättslig grund för behandlingen

Personuppgifter behandlas huvudsakligen med stöd av:

  • ✓berättigat intresse, till exempel för att kunna svara på en förfrågan och hantera kontakt
  • ✓avtal eller åtgärder inför avtal, till exempel när du vill beställa en tjänst eller be om offert
  • ✓rättslig förpliktelse, till exempel när uppgifter behöver sparas enligt bokföringsregler

Om någon behandling i framtiden skulle kräva samtycke används det i så fall för just den delen.

5. Hur länge uppgifterna sparas

Uppgifter sparas inte längre än nödvändigt för det ändamål de samlades in för.

  • ✓förfrågningar som inte leder till uppdrag sparas normalt bara så länge det behövs för kontakt och uppföljning
  • ✓uppgifter kopplade till genomförda uppdrag kan sparas längre om det behövs för administration, support, överlämning eller bokföring
  • ✓bokföringsunderlag sparas enligt de krav som gäller enligt lag

Om du vill fråga vad som finns sparat eller be om radering där det är möjligt, hör av dig.

6. Delning av uppgifter och använda verktyg

Personuppgifter säljs inte vidare. Uppgifter delas inte heller med andra i onödan. Däremot kan vissa tekniska tjänster användas för att sidan och kontakten ska fungera i praktiken.

Exempel på sådana tjänster kan vara:

  • ✓formulärtjänst för att ta emot formulärmeddelanden
  • ✓e-posttjänst för att läsa och besvara förfrågningar
  • ✓webbhotell eller domänrelaterade tjänster
  • ✓betal- eller fakturarelaterade tjänster när ett uppdrag genomförs

Om du använder formulären på sidan kan uppgifterna alltså passera via den tekniska formulärtjänst som används för att skicka meddelandet vidare till min e-post.

Om ett uppdrag kräver det kan även teknisk information behandlas för att bygga, uppdatera eller lämna över hemsidan eller webblösningen på ett tydligt sätt.

Om du vill minimera vad du skickar via formulär går det alltid bra att först skriva kortfattat och ta resten i nästa steg.

7. Dina rättigheter

Du har enligt dataskyddsreglerna rätt att i vissa fall:

  • ✓få information om vilka personuppgifter som behandlas
  • ✓begära rättelse av felaktiga uppgifter
  • ✓begära radering av uppgifter när det är möjligt
  • ✓invända mot viss behandling
  • ✓begära begränsning av behandling i vissa situationer

Om du vill använda någon av dina rättigheter kontaktar du mig via e-post. Du har också rätt att lämna klagomål till Integritetsskyddsmyndigheten om du anser att personuppgifter behandlas fel.

8. Säkerhet

Rimliga tekniska och organisatoriska åtgärder används för att skydda personuppgifter mot obehörig åtkomst, förlust eller missbruk.

Samtidigt är ingen vanlig webbplats, e-post eller formulärtjänst en magisk bunker. Därför är det klokt att inte skicka mer persondata än vad som faktiskt behövs.

Det gäller även material, inloggningsuppgifter och annan teknisk information som ibland behövs i uppdrag där jag bygger eller uppdaterar hemsidor och webblösningar.
